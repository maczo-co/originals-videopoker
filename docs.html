maczo Originals — Video Poker · Provably Fair

Open source · MIT · offline · zero-dependency. The crypto core is the shared engine originals-verify; this game adds the pure Video Poker resolver. Full pages: wiki.

The engine (frozen)

Every Originals game draws its randomness from one frozen recipe — uint32 words (this differs from Lucky Spin, which uses uint64 words + rejection sampling):

commitment = SHA-256(serverSeed)                         # lowercase hex, shown before any bet
for cursor = 0, 1, 2, …:                                 # advance when more words are needed
    digest = HMAC-SHA256(key = serverSeed, msg = `${clientSeed}:${nonce}:${cursor}`)
    consume 4 bytes at a time, big-endian, as uint32 words   # u in [0, 2^32)

Settlement compares integers derived from the words — no floating point. A payout is floor(bet × multiplier_e8 / 1e8).

Video Poker's resolve step bias: uniform

deck = shuffle(52); deal order[:5]; holds keep, others draw from order[5:]; final 5-card hand pays payoutMultiplier[category]× (multiplier_e8 = that × 1e8).

Bias class uniform — maps words without a biased modulo reduction (bit extraction, or a uniform Fisher-Yates / without-replacement draw) — no modulo bias.

Three ways to verify

  1. One click. After a seed is revealed, each bet deep-links to the shared verifier: verify.maczo.co/originals/?game=videopoker&server=…&commit=…&client=…&nonce=…&params=… — or open this repo's index.html (self-contained, offline).
  2. Browser, manual. Open index.html, paste your serverSeed / clientSeed / nonce / params, press Verify. View source — nothing leaves your machine.
  3. Node CLI. git clone https://github.com/maczo-co/originals-videopoker && cd originals-videopoker && npm install && npm test reproduces the bundled known-answer vectors (JS ⇄ the server's authoritative Python).

Published odds

The odds are published at paytable.json — the resolver only indexes into it, so the odds you see are the odds that settled your bet.

Reveal

Before the seed is revealed the verifier can still check the commitment; it cannot reproduce the outcome (the deep-link shows “awaiting reveal”). After you rotate your seed, the revealed serverSeed is the exact HMAC key and the full outcome reproduces offline.
