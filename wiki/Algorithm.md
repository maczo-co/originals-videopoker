# The Algorithm

**Video Poker**'s outcome is fully deterministic: the same `(serverSeed, clientSeed, nonce)` plus the
published paytable always produce the same result. The verifier in this repo reproduces it byte-for-byte,
and its known-answer vectors were generated by the operator's authoritative server code.

## 1. Commitment

Before your first bet on a seed, the site shows:

```
commitment = SHA-256(serverSeed)      // 64 hex chars, lowercase
```

The raw `serverSeed` stays secret while it is active. The commitment proves the seed already existed and
cannot be swapped later — any change would produce a different SHA-256.

## 2. Per-bet randomness (the shared engine)

```
digest = HMAC-SHA256(key = serverSeed, message = "clientSeed:nonce:cursor")   // cursor starts at 0
```

The 32-byte digest is consumed **4 bytes at a time, big-endian, as `uint32` words** (`u` in `[0, 2^32)`);
the `cursor` increments and the HMAC re-runs whenever a game needs more words than one digest holds.

> This is the **maczo Originals** engine — `uint32` (4-byte) words. It differs from **Lucky Spin**, which
> uses 8-byte `uint64` words + rejection sampling. Do not mix the two.

The engine lives once, in [`@maczo/originals-verify`](https://github.com/maczo-co/originals-verify); this
repo only adds Video Poker's resolve step on top of the word stream.

## 3. Video Poker's resolve step

deck = shuffle(52); deal order[:5]; holds keep, others draw from order[5:]; final 5-card hand pays payoutMultiplier[category]× (multiplier_e8 = that × 1e8).

**Bias class: `uniform`** — maps words without a biased modulo reduction (bit extraction, or a uniform Fisher-Yates / without-replacement draw) — no modulo bias.

Settlement compares integers derived from the words (no floating point). Money is integer minor units; a
payout is `floor(bet × multiplier_e8 / 1e8)`.

## 4. Reveal

When you **Rotate** your seed, the site permanently retires it and reveals the original `serverSeed`. You
can then check `SHA-256(serverSeed) == commitment` and re-run every past Video Poker bet under that seed.

## Reference implementations (this repo)

- `resolve.js` — the pure Video Poker resolver (mirrors the server's `libs/game_math/videopoker.py`)
- `verify.js` — Node CLI + `npm test` (`--selftest`) against `test-vectors.json`
- `paytable.json` — the published odds the resolver indexes into (never recomputed)

`resolve.js` imports the shared engine and uses `BigInt` for exact integer math.
